配置文件详解：openclaw.json

你最该掌握的一页：知道每个字段做什么，排错速度会快很多。

📍文件位置

macOS / Linux

~/.openclaw/openclaw.json

Windows

%USERPROFILE%\.openclaw\openclaw.json

🧱最小可用配置（示例）

json
{ "providers": { "openai": { "apiKey": "${OPENAI_API_KEY}" } }, "defaultModel": "openai/gpt-5", "channels": { "telegram": { "enabled": true, "botToken": "${TELEGRAM_BOT_TOKEN}", "dmPolicy": "pairing", "groups": { "*": { "requireMention": true } } } } }

🧠Providers 字段要点

  • apiKey：模型供应商密钥，建议使用环境变量注入。
  • baseURL：使用代理/兼容网关时填写。
  • defaultModel：默认模型，建议使用稳定版本名。

📡Channels 字段要点

  • enabled：是否启用渠道。
  • dmPolicy：私聊策略（如 pairing）。
  • groups.*.requireMention：群聊是否必须 @ 才触发。
  • groupPolicy：允许名单/策略控制。
🔐
建议默认开启 requireMention
避免在群聊中被噪声触发，降低误调用成本。

🛡️安全实践清单

  • 密钥不要硬编码到仓库，统一从环境变量读取。
  • 公开部署时收紧网络入口，关闭不必要端口。
  • 定期轮换密钥并清理历史 token。
  • 日志与备份先脱敏再上传。
