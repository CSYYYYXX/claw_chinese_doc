常见问题

遇到问题？这里可能有答案。

📦 安装问题

npm install 时报错 "permission denied" 或 "EACCES" +

这是 npm 权限问题，常见于 macOS 和 Linux。

解决方案 1：使用 sudo（简单但不推荐）

sudo npm install -g openclaw@latest

解决方案 2：修复 npm 权限（推荐）

mkdir ~/.npm-global npm config set prefix '~/.npm-global' # 然后把 ~/.npm-global/bin 加到 PATH 环境变量

解决方案 3：使用 nvm 管理 Node.js

nvm 安装的 Node.js 默认不需要 sudo。

安装成功但运行 openclaw 提示"命令不存在" +

通常是 PATH 环境变量没有包含 npm 全局安装目录。

检查方法：

npm config get prefix

把输出的 bin 子目录加到 PATH。

示例（macOS/Linux，假设 npm prefix 是 /usr/local）：

echo 'export PATH="/usr/local/bin:$PATH"' >> ~/.zshrc source ~/.zshrc

Windows 用户：重新安装 Node.js，确保勾选 "Add to PATH" 选项，然后重启终端。

Windows 上安装很慢或超时 +

可能是网络问题，尝试使用国内镜像：

npm config set registry https://registry.npmmirror.com npm install -g openclaw@latest

安装完成后可以恢复官方源：

npm config set registry https://registry.npmjs.org

⚙️ 配置问题

API Key 填了但模型不工作 +

检查以下几点：

  1. API Key 格式：OpenAI 的 Key 以 sk- 开头，Anthropic 的 Key 也是固定前缀
  2. 账户余额：确认 API 账户有余额/额度
  3. 网络访问：如果在国内使用 OpenAI，需要配置代理
  4. 模型名称：检查配置文件中的模型名是否正确

配置代理（国内用户）：

# 设置环境变量 export HTTP_PROXY=http://127.0.0.1:7890 export HTTPS_PROXY=http://127.0.0.1:7890

或者在配置文件中设置：

{ "providers": { "openai": { "apiKey": "sk-xxx", "baseURL": "https://your-proxy.com/v1" } } }
如何配置多个 AI 模型并切换？ +

OpenClaw 支持配置多个 provider，然后在对话中切换模型。

配置示例：

{ "providers": { "openai": { "apiKey": "sk-xxx" }, "anthropic": { "apiKey": "sk-ant-xxx" }, "google": { "apiKey": "xxx" } }, "defaultModel": "openai-codex/gpt-5.3-codex" }

在对话中切换模型：

发送 /model anthropic/claude-3-7-sonnet 即可切换当前会话的模型。

配置文件在哪里？ +

配置文件位置：

  • macOS / Linux：~/.openclaw/config.json
  • Windows：C:\Users\你的用户名\.openclaw\config.json

直接用文本编辑器打开即可修改。

# macOS/Linux code ~/.openclaw/config.json # Windows (PowerShell) code $env:USERPROFILE\.openclaw\config.json

💬 使用问题

AI 回复一半就停了 +

可能的原因：

  1. 达到 token 限制：模型单次输出有上限，可以发送"继续"让 AI 继续输出
  2. 网络中断：检查网络连接，尤其是使用代理时
  3. API 超时：较长的回复可能超时，尝试在配置中增加超时时间
💡
大多数情况下，回复"继续"或"请继续"就能让 AI 补全剩余内容。
如何查看对话历史？ +

OpenClaw 会自动保存对话历史：

  • Web Chat：历史记录在侧边栏
  • Telegram/Discord：直接在聊天记录中查看
  • 命令行：openclaw sessions list 查看会话列表

导出对话：

openclaw sessions export > conversation.json
如何让 AI 记住一些信息？ +

OpenClaw 支持"记忆"功能，让 AI 在不同会话中记住重要信息。

方式一：在对话中告诉 AI

直接说"记住这个：我喜欢用 Python 写代码"，AI 会写入记忆文件。

方式二：编辑记忆文件

记忆文件位于工作目录下：

  • MEMORY.md - 长期记忆
  • memory/YYYY-MM-DD.md - 每日记录
AI 总是忘记之前说过的话 +

这通常是因为上下文窗口满了。AI 只能"看到"最近的 N 条消息。

解决方案：

  1. 开始新会话：发送 /new 清空上下文，重新开始
  2. 总结要点：让 AI 先总结之前的讨论，然后在新会话中继续
  3. 使用记忆功能：把重要信息写入 MEMORY.md

🔧 高级问题

如何安装 Skills（技能包）？ +

Skills 是 OpenClaw 的扩展能力包，可以从 ClawHub 安装：

# 搜索技能 openclaw skill search <关键词> # 安装技能 openclaw skill install <skill-name> # 列出已安装技能 openclaw skill list

或者访问 ClawHub 浏览所有可用技能。

如何让 AI 控制我的电脑（执行命令、操作文件）？ +

OpenClaw 默认就支持文件操作和命令执行，但需要注意安全设置。

安全模式（默认）：

  • AI 可以读取文件
  • 写入文件需要确认
  • 执行命令需要确认

调整安全级别：

{ "security": { "mode": "full" // allowlist | full } }
⚠️
谨慎使用 full 模式，AI 将可以执行任意命令而无需确认。
Gateway 占用太多内存怎么办？ +

如果长时间运行，可以定期重启 Gateway 释放内存。

使用 systemd（Linux）自动管理：

openclaw onboard --install-daemon # 查看状态 systemctl status openclaw-gateway # 重启 sudo systemctl restart openclaw-gateway

使用 pm2（跨平台）：

npm install -g pm2 pm2 start "openclaw gateway" --name openclaw pm2 save pm2 startup
如何部署到云服务器？ +

OpenClaw 可以部署到任何支持 Node.js 的服务器上。

基本步骤：

  1. 在服务器上安装 Node.js
  2. npm install -g openclaw@latest
  3. 配置 ~/.openclaw/config.json
  4. 运行 openclaw onboard --install-daemon 安装为系统服务

注意事项：

  • 确保服务器防火墙开放对应端口（如果使用 Web Chat）
  • 建议配置 HTTPS（使用 nginx 反向代理 + Let's Encrypt）
  • 定期备份数据目录
