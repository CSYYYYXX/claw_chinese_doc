快速上手

先选模型，再接渠道，最后发第一条消息。照着做，10 分钟可跑通。

通义 / 豆包 / DeepSeek

OpenAI 详细配置

  1. 去 OpenAI 平台创建 API Key。
  2. 运行 openclaw onboard，供应商选 OpenAI。
  3. 模型先选 openai/gpt-5。
  4. 执行 openclaw status 确认模型已生效。

Anthropic 详细配置

  1. 在 Anthropic 控制台生成 API Key。
  2. 运行 openclaw onboard 选择 Anthropic。
  3. 模型建议 anthropic/claude-sonnet-latest。
  4. 如遇速率限制，切换备用模型或降低输出长度。

Google 详细配置

  1. 在 Google AI Studio 创建 API Key。
  2. 运行 openclaw onboard，供应商选 Google。
  3. 模型优先 Gemini 2.5 系列。
  4. 若地区受限，先用 OpenAI/Anthropic 跑通再回切。

国内模型详细配置

  1. 先确定供应商（通义/豆包/DeepSeek）。
  2. 准备 API Key 与 base URL。
  3. 在向导中选择兼容模式，或手动编辑 ~/.openclaw/openclaw.json。
  4. 用简单问答验证鉴权与网络，再投入正式使用。

🧭官方推荐的稳定用法

1. 用深度状态确认运行质量

命令行
openclaw status --deep openclaw doctor

如果这里有红色/错误项，先修复再继续配渠道，能减少 80% 的后续问题。

2. 安装为后台服务（长期运行）

命令行
openclaw onboard --install-daemon openclaw gateway status

适合 VPS 或长期在线场景，避免终端断开后服务停止。

3. 统一入口：Dashboard + 日志跟踪

命令行
openclaw dashboard openclaw logs --follow

Dashboard 看整体状态，日志看实时错误，两者配合排查最快。
