1️⃣第 1 章：准备环境

最低建议配置

  • 个人本地体验：4 核 CPU / 8GB RAM / 稳定网络。
  • 云端常驻：1 vCPU / 2GB RAM 起步；多渠道建议 2 vCPU / 4GB RAM。
  • 系统建议：Windows 11 + WSL2、macOS 13+、Ubuntu 22.04+。

Node.js 版本建议

建议使用 Node.js LTS，优先 22+。安装后先验证：

bash
node -v npm -v

2️⃣第 2 章：首次安装（保姆式）

第 1 步：安装 OpenClaw

bash
npm install -g openclaw@latest openclaw --version

第 2 步：运行向导

bash
openclaw onboard

向导会引导你选择模型、填 API Key、接渠道、设置网关。

第 3 步：安装后台服务（推荐）

bash
openclaw onboard --install-daemon openclaw gateway status

这一步能保证终端关闭后服务仍在线。

📂
配置文件位置（重要）
macOS/Linux: ~/.openclaw/openclaw.json；Windows: %USERPROFILE%\.openclaw\openclaw.json

3️⃣第 3 章：接入渠道与模型

渠道优先级建议

新手首选 Telegram；团队协作常用 Discord/Slack；移动端重度用户可选 WhatsApp。

模型选择建议（截至 2026-03-05）

OpenAI/Anthropic/Google 都可用。Google 侧建议优先 Gemini 2.5 系列。

避免群聊刷屏

群里默认开启 mention 触发（requireMention），只在被 @ 时响应。

🔑
Key 管理
API Key 和 Bot Token 不要写进公开仓库；优先用环境变量或密钥管理工具。

4️⃣第 4 章：每天会用到的命令

查看运行状态

bash
openclaw status openclaw status --deep

系统体检

bash
openclaw doctor

实时日志

bash
openclaw logs --follow

可视化面板

bash
openclaw dashboard

5️⃣第 5 章：故障树（按症状排查）

症状 A：Bot 在线但不回复

先看群权限与 mention 规则，再看 logs --follow 是否有 401/403/429。

症状 B：模型调用失败

检查 API Key、额度、base URL、地区网络代理，然后跑 doctor。

症状 C：重启后服务消失

确认已执行 onboard --install-daemon 并检查网关服务状态。

6️⃣第 6 章：安全与生产化

  • Dashboard 只对内网开放；公网访问使用反向代理与认证。
  • 最小权限原则：渠道机器人只给需要的权限。
  • 定期轮换 API Key / Bot Token，及时撤销旧凭证。
  • 日志中避免泄露敏感信息；备份时对配置做脱敏。
  • 生产环境优先采用 VPS + daemon + 监控告警。
