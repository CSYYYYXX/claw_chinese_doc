云部署（生产可用版）

面向 VPS/云服务器的实战部署，包含官方推荐命令、服务常驻方案和安全基线。

☁️部署前决策

适合场景

需要 24x7 在线回复、多人共享 Bot、跨设备持续会话时，建议部署到 VPS。

最低建议配置

1 vCPU / 2 GB RAM / 20 GB SSD 起步；多渠道并发建议 2 vCPU / 4 GB RAM。

系统建议

Ubuntu 22.04+ 或 Debian 12+，Node.js 使用 LTS 且建议 22+。

🚀标准部署流程

1) 安装 Node.js 与 OpenClaw

bash
npm install -g openclaw@latest openclaw --version

2) 运行配置向导并安装后台服务

bash
openclaw onboard --install-daemon

向导完成后会配置模型、渠道与服务常驻参数。

3) 检查服务状态与深度体检

bash
openclaw gateway status openclaw status --deep openclaw doctor

4) 启动可视化面板并观察日志

bash
openclaw dashboard openclaw logs --follow

🔐生产安全基线

  • Dashboard 仅绑定内网或回环地址，公网访问请走反向代理和鉴权。
  • Bot Token/API Key 只放环境变量或密钥管理系统，避免写入仓库。
  • 渠道默认开启 mention 触发（如 requireMention），防止群聊噪声触发。
  • 限制来源网络（防火墙/安全组/Tailscale/Zero Trust）。
  • 定期轮换密钥并清理不再使用的渠道授权。
⚠️
高风险操作提醒
不要把 ~/.openclaw 与包含密钥的配置文件直接公开上传。

🧰运维常用命令

服务生命周期

openclaw gateway start
openclaw gateway stop
openclaw gateway restart

状态与诊断

openclaw status
openclaw status --deep
openclaw doctor

实时日志

openclaw logs --follow
结合 Dashboard 观察异常链路。

🧩常见故障速排

Bot 在线但不回复

先查权限、mention 策略和日志；多数是权限或频道策略问题。

bash
openclaw logs --follow

重启后服务失联

确认是否执行过 onboard --install-daemon，并检查网关状态。

bash
openclaw gateway status

模型调用频繁失败

重点检查 API Key、配额、地区网络与 base URL 配置。
